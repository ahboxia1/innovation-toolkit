# 创新脚手架 · Innovation Toolkit

**三步，从模糊想法到能立项的一页纸。**

基于 Stanford d.school 设计思维（Design Thinking）前三阶段，为 Claude Code 环境蒸馏。

`Design Thinking`（设计思维） `AI Native`（AI原生） `Claude Code Skill`（技能）

## 为什么需要这个？

**这是个思维框架，不只是工具。**

以前怎么创新？人做研究 → 人写文档 → 人做PPT → 人汇报 → 人被质疑。AI顶多最后帮你润色一下措辞。

AI native（AI原生）的做法不一样：**从第一天就跟AI一起想、一起写、一起迭代**。三次对话下来，你就习惯了这种工作方式。

这工具包干一件事：**帮员工从模糊想法走到能立项的一页纸。**

摸清战场、定义问题、想出方案，三步搞定。最后交出来的是一张**立项一页纸**——DT有DT的模板（AI季度探索·7项一页纸），千丁有千丁的模板（蜂巢/敏捷立项·8项商业计划书），对号入座，拿去就能申报。创意走到这一步，才有机会变成商业价值。

## 方法论：Stanford d.school 设计思维

底子是 [Stanford d.school](https://dschool.stanford.edu/) 的设计思维（Design Thinking Bootleg, 2018）——斯坦福哈索·普拉特纳设计学院出的创新方法论。

d.school 2005年由 IDEO 创始人 David Kelley 在斯坦福创办，SAP联合创始人 Hasso Plattner 捐了3500万美元冠名。它那套五阶段模型基本是全球企业创新的事实标准——IBM 拿它搭了 Enterprise Design Thinking 体系培训几万人，Google Ventures 的 Design Sprint 也是从这里长出来的，德国波茨坦甚至复刻了一整个 d.school。

d.school把创新拆成五个阶段，但这五步不是瀑布流——你可以跳着走，也可以回退。我们只做前三步（Empathize → Define → Ideate），后面两步（Prototype → Test）靠具体场景里的数字员工和真实用户验证。

## 三步流程

```
  ┌──────────┐    ┌──────────┐    ┌──────────┐
  │ EMPATHIZE│───▶│  DEFINE  │───▶│  IDEATE  │
  │  (共情)   │    │  (定义)   │    │  (孵化)   │
  └──────────┘    └──────────┘    └──────────┘
   Step 1          Step 2          Step 3
   行业研究         定义问题         发散方案
```

d.school自己说了，这五步不是死板的线性流程——Define的时候可能发现得回退重新Empathize，Ideate的时候可能发现POV（Point of View，观点陈述）定义偏了。随时跳，随时回。

---

### Step 0 — 引导（不知道从哪开始？）

正常，很多人这时候脑子是空的。系统不会上来就问"你想研究什么"——先聊两句，了解你是谁：

**先了解你：**
1. **你在什么团队/岗位？** — 技术产品运营市场HR财务，做什么工作
2. **你日常工作主要在忙什么？** — AI先摸清你的业务语境

**然后给方向：** AI根据你的职能和日常工作，suggest 2-3个大方向——"你这个岗位通常会关注XX领域的创新机会，要不要了解一下？"

**再收窄到具体问题：**
3. **你最近有没有注意到什么「客户在抱怨但没人解决」的事？**
4. **如果让你做一件事让客户愿意付费，你觉得是什么？**

从"你是谁"到"大方向"到"具体问题"，一步一步引出来。核心原则：**把员工往客户痛点和商业机会方向引导。**

---

### Step 1 — Empathize / Research（共情 · 行业研究）

> *"Empathy is the foundation of human-centered design."（共情是以人为本设计的基石）— d.school Bootleg*

这步干什么：**快速摸清一个赛道**。出一千字的行业扫描判断"这个方向值不值得继续往下走"。

d.school说共情有三种做法：Observe（观察用户行为）、Engage（跟用户对话访谈）、Immerse（穿上用户的鞋走一英里）——都是感性层面的用户感知。在感性之外，Step 1额外加了理性信息层——联网搜索 + Playwright爬虫，帮你在感性认知之外补上真实的市场数据。

确认方向后，AI联网做双轴调研：纵轴追时间线（发展历程→关键节点→当前格局），横轴追竞品（谁在做→差异在哪→用户怎么说）。需要深挖的话启动Playwright爬虫。最后输出一份千字级HTML行业扫描报告。

---

### Step 2 — Define（定义 · 问题定义）

> *"The define mode is when you unpack your empathy findings into needs and insights and scope a meaningful challenge."（定义阶段是将共情发现拆解为需求和洞察，界定有意义的挑战）— d.school Bootleg*

这步干什么：**把调研结果拆明白，形成POV（Point of View，观点陈述）。**

Define不是简单说"问题是什么"——它要你回答：你在解决**谁的什么问题**，为什么这个问题**值得解决**，什么是**颠覆性（game-changing）**的方向。

**POV框架：**
```
1. We met: [你遇到了谁——具体的人，不是笼统的群体]
2. We were surprised to notice: [你惊讶地发现了什么——反直觉的观察]
3. We wonder if this means: [你猜测这意味着什么——深层需求]
4. It would be game-changing to: [如果能怎样将是颠覆性的——方向]
```

AI帮你从调研里提取关键发现，提炼needs（需求）和insights（洞察），形成POV，然后从POV推导出HMW（How Might We，我们不妨怎样）问题——这就是Step 3发散方案的起点。

**到这里会过一道关：方向验证gate。**

POV出来之后、进Step 3之前，四巨头先帮你快审一下方向。

四巨头是四个虚拟评审官，分别蒸馏了巴菲特、马斯克、纳瓦尔、Karpathy 的思维方式。每个人盯着一个维度——护城河、第一性原理、杠杆、数据飞轮。目的很简单：趁还没花时间想方案，先用四个商业视角快速筛一遍，方向不行趁早打回。

| 评审官 | 他会问你什么 |
|--------|------------|
| 巴菲特 | 护城河在哪——这个问题被解决了，别人能轻易复制吗？ |
| 马斯克 | 这问题该存在吗——你是解决真问题，还是在优化一个不该存在的流程？ |
| 纳瓦尔 | 杠杆够不够——解决这个问题需要谁的许可？能无需许可地扩展吗？ |
| Karpathy | 有数据飞轮吗——这个问题有没有越做越好、数据越积越多的潜力？ |

✅ 多就过，❌ 多就打回重新定义问题，有分歧就跟你聊。

**gate放这里是有原因的。** 万一你的POV指向的是"内部效率工具"，纳瓦尔直接会说"没有无需许可的杠杆"，巴菲特会说"零护城河"——趁早打回，比做完一页纸再发现方向不对强太多。

---

### Step 3 — Ideate（孵化 · 方案发散与收敛）

> *"Ideate is the mode in which you generate radical design alternatives — a mode of 'flaring' instead of 'focus'."（构思阶段用来生成大胆的、突破性的备选方案——先打开思路，不要急着收拢）— d.school Bootleg*

这步干什么：**基于POV和HMW问题发散方案，收敛到一张立项一页纸。**

Ideate不是漫无目的地想idea——问题已经定义清楚了，现在对着这个问题想方案。

1. **发散（Go Wide）**：每个HMW问题至少3个方案，施加约束（Constraints）激发创意
2. **收敛（Narrow In）**：你选出最感兴趣的2-3个
3. **深化（Describe Your Concept）**：方案叫什么、为谁做、做什么
4. **输出一页纸**：DT走7项模板，千丁走8项模板

**输出前要过段永平的关：**
- 你是在解决真问题，还是在找问题来配方案？
- 核心价值说得出数字吗？（"提升效率"→过不了，"缩短3天→1天"→可以）
- 失败了最大的风险是什么？
- 有没有"不该做但为了创新而做"的部分？

过不了就说，不帮你粉饰。

## 快速开始

### 环境要求

- **Claude Code** 已安装并可用
- **Node.js** >= 18（Step 1 行业研究的爬虫需要）

### 安装

**Windows：**
```bat
install.bat
```

**Mac / Linux：**
```bash
chmod +x install.sh
./install.sh
```

### 验证

```bash
node scripts/check-env.mjs
```

### 使用

打开 Claude Code，说：

```
创新工具包
```

或者直接跳到任意步骤：
```
帮我调研一下智能家居赛道    # → Step 1 Empathize（共情）
帮我把问题说清楚             # → Step 2 Define（定义）
帮我想方案 / 写立项一页纸    # → Step 3 Ideate（孵化）
```

## 可选附加模块

核心三步之外的，需要时单独调用：

| 模块 | 干什么 | 怎么触发 |
|------|--------|---------|
| **04-pitch** | 把一页纸变成HTML路演展示页，3种风格可选 | 「做个路演展示」 |
| **05-review** | 四巨头完整版评审，每人200-400字深度分析 + 雷达图 | 「帮我审一下这个idea」 |
| **06-knowledge-base** | 组织创新记忆，记历史idea避免重复造轮子 | 「历史idea」「查一下做过类似的」 |

Step 2里的方向验证gate用的是轻量版四巨头（一人一句话快审），05-review是完整版。
