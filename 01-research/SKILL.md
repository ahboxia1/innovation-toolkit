# 创新行业研究

帮员工快速搞懂一个赛道/公司/产品，判断值不值得创新投入。

不是万字研究报告，是**千字级行业扫描**——足够判断"这个方向值不值得深入"。

## 触发判断

| 用户状态 | 走哪条路 |
|---------|---------|
| 给了具体公司/产品名 | → 直接研究 |
| 给了赛道/领域名 | → 先找代表玩家再研究 |
| 只说"帮我找个方向" | → 引导用户说清楚感兴趣的领域 |

---

## 研究流程

### Step 1：明确研究对象

确认三件事（用户已经说清楚就跳过）：
1. **研究对象**：公司名/产品名/赛道名
2. **研究动机**：为什么要了解？（找创新机会？评估竞品？了解行业？）
3. **特别关注**：有没有特别想深入的点？

### Step 2：双轴信息收集

**必须联网搜索，不能只靠已有知识。**

用子Agent并行收集（如果环境支持），否则按顺序搜索：

**纵轴（时间线）**：
- 起源：什么时候出现？为什么？
- 关键节点：重大版本/融资/战略转向/危机
- 现状：当前格局如何？

**横轴（竞品对比）**：
- 主要竞品有哪些（最多5个）
- 各自的核心差异是什么
- 用户口碑（优先一手来源：GitHub Issues、知乎、Reddit）

### Step 3：深度抓取（可选）

当用户需要某个网站/竞品官网的完整内容时，使用Playwright爬虫：

```bash
node "SKILL_DIR/spider.mjs" --url https://目标网站.com --out /tmp/research-output
```

要求：
- Playwright已安装（`npm install playwright`）
- 需要登录态时加 `--login` 参数
- 抓取结果保存在临时目录，用完即删

### Step 4：输出行业扫描

千字以内，结构如下：

```
## XX 行业扫描

**一句话定义**：用一句话说清楚这个赛道是什么

### 纵轴：发展脉络
[300-500字，关键节点+因果逻辑，不是流水账]

### 横轴：竞争图谱
[300-500字，主要玩家+核心差异]

### 洞察判断
[200-300字]
- 这个赛道最大的机会在哪
- 最大的坑是什么
- 如果要创新，切入点建议
```

## 信息来源优先级

一手来源 > 二手来源，避免循环印证：

| 信息类型 | 优先来源 |
|---------|---------|
| 产品/技术 | 官方博客、GitHub、创始人公开言论 |
| 商业数据 | 官方公告、工商/SEC文件 |
| 用户口碑 | 社区讨论（知乎、Reddit、GitHub Issues） |
| 行业分析 | 权威媒体原创报道（非转载） |

搜不到的信息标注「暂缺」，不编造。

---

## 输出规范

- 每段不超过5行
- 有判断有态度：觉得有潜力就说，觉得是坑也直说
- 关键数字标注来源
- 不用AI腔（不说"综上所述""值得注意的是"）
- 不用emoji（除非用户主动用）

---

## 后续衔接

输出扫描报告后，提示用户：
> 行业摸清了。下一步可以进入「创意孵化」——从痛点里长出创新idea。
